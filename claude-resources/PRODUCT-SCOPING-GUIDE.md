## 🎯 这个技能是什么？

**product-scoping** 是一个**通用的产品需求预处理技能**，用于在生成PRD之前，通过结构化的4步决策漏斗来澄清复杂和模糊的产品需求。

### 核心特点

1. **通用性** - 不仅限于AI产品，适用于所有复杂产品
2. **结构化** - 4步决策漏斗，系统性澄清需求
3. **资产导向** - 必须盘点现有资源，避免空中楼阁
4. **预处理** - 在 requirement-to-prd 之前运行

---

## 📋 4步决策漏斗

### 阶段一：定义"谁"与"痛点"
```
Q1: 核心用户是谁？
    - 泛人群 vs 行业人群 vs 特殊需求 vs 企业用户

Q2: 数据源的深度？
    - 手动录入 vs 设备采集 vs 系统打通 vs 第三方

Q3: 第一痛点是什么？
    - 理解困难 vs 缺乏监督 vs 突发恐慌 vs 效率低下 vs 决策困难
```

### 阶段二：定义"核心能力"与"边界"
```
Q4: 核心能力的深度？
    【AI产品】翻译解释 vs 关联分析 vs 预测演练 vs 自动执行
    【平台产品】信息聚合 vs 流程自动化 vs 协同连接 vs 数据分析
    【工具产品】效率提升 vs 质量保证 vs 标准化

Q5: 干预边界在哪里？
    - 保守提示 vs 智能建议 vs 主动干预

Q6: 主动管理的手段？
    - 强提醒 vs 游戏化 vs 场景触发 vs 社交压力 vs 不需要
```

### 阶段三：定义"形态"与"闭环"
```
Q7: 产品第一界面？
    - Chat-First（对话流）
    - Task-First（任务列表）
    - Dashboard-First（数据看板）
    - Feed-First（信息流）
    - 混合卡片

Q8: 激励机制？
    - 精神荣誉 vs 实体权益 vs 功能解锁 vs 费率挂钩 vs 社交认可

Q9: 人工兜底模式？
    - 无兜底 vs 外部轻服务 vs 自营深度服务 vs 混合模式
```

### 阶段四：资产盘点与差异分析 ⚠️ 最关键
```
Q10: 数据底座现状？
     As-Is: 静态数据 vs 有接口 vs 双向读写 vs 实时同步
     To-Be: 期望达到什么状态
     Gap: 差距是什么

Q11: 核心能力现状？
     【AI产品】空白 vs 规则库 vs 通用模型 vs 垂类模型
     【平台产品】空白 vs 基础系统 vs 成熟系统 vs 生态系统
     As-Is / To-Be / Gap

Q12: 服务团队现状？
     无团队 vs 简易客服 vs 专业团队 vs 成熟系统
     As-Is / To-Be / Gap

Q13: 前端载体现状？
     从零开发 vs 旧系统改造 vs 嵌入式 vs 多端并行
     As-Is / To-Be / Gap
```

## ⚠️ 重要提示

### 阶段四最关键！

阶段四（资产盘点与差异分析）是**最重要**的阶段：

**为什么重要？**
- 识别现有资源和能力
- 明确差距和风险
- 避免"空中楼阁"方案
- 提供可行的实施路径

**永远不要跳过阶段四！**

### 差异分析模板

每个维度都要回答：
```
当前状态 (As-Is)：我们现在有什么
期望状态 (To-Be)：我们想要什么
差距 (Gap)：需要做什么才能达到
```
