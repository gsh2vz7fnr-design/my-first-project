### 1. **requirement-to-prd** ⭐⭐⭐⭐⭐
**核心PRD生成技能**

**功能**：
- 将业务需求转换为结构化PRD文档
- 专为金融/医疗合规产品设计
- 自动生成Mermaid流程图
- 包含数据模型设计
- v2.0版本，友好的交互方式

**触发关键词**：
- "写PRD"、"产品需求文档"
- "业务需求"、"功能需求"
- "金融"、"医疗"、"合规"

**工作流程**：
```
Phase 0: 收集上下文
  ├─ 目标用户是谁？
  ├─ 什么平台？（Web/App/小程序）
  └─ 有敏感数据吗？

Phase 1: 生成大纲
  └─ 等待你确认

Phase 2: 生成完整PRD
  ├─ 需求概述
  ├─ 功能列表（管道分隔格式）
  ├─ 数据字段汇总
  ├─ 埋点逻辑
  └─ 待确认事项

Phase 3: 生成流程图
  └─ Mermaid格式流程图
```

**特色功能**：
- 🔍 自动推断隐含规则
  - 搜索场景：历史记录、搜索建议、热门搜索、排序、无结果处理、防抖
  - 筛选场景：组合逻辑、结果计数、重置、持久化、无结果处理
  - 上传场景：格式限制、大小限制、进度、重试、预览、病毒检测
- 📊 管道分隔格式（易读易维护）
- 🔒 自动识别敏感数据和合规要求
- 📈 包含埋点逻辑设计

### 2. **prototype-to-prd** ⭐⭐⭐⭐
**从UI原型生成PRD**

**功能**：
- 从UI截图/原型生成基础PRD
- 明确区分"可见"和"隐藏"需求
- 双钻石规范流程的第一阶段

**触发关键词**：
- "这是我的设计稿"、"UI原型"
- "从截图生成PRD"
- "原型转PRD"

**工作流程**：
```
上传UI截图
  ↓
⛔ STOP RULE: 必须回答3个P0问题
  ├─ 目标用户是谁？
  ├─ 什么平台？
  └─ 有敏感数据吗？
  ↓
视觉分析（可见元素）
  ├─ UI组件
  ├─ 文案
  └─ 导航
  ↓
逻辑推断（隐藏规则）
  ├─ 验证规则
  ├─ 状态管理
  ├─ 错误处理
  └─ 限制条件
  ↓
基础PRD + 待确认问题
```

**分析框架**：
| 层级 | 分析内容 | 示例 |
|------|---------|------|
| **可见** | UI组件、文案、导航 | "登录按钮"、"忘记密码链接" |
| **隐藏** | 验证、状态、错误、限制 | "密码复杂度规则"、"5次失败后锁定" |
| **业务** | 规则、权限、数据流 | "只有管理员能看到"、"数据同步到CRM" |

### 3. **doc-coauthoring** ⭐⭐⭐
**结构化文档协作**

**功能**：
- 结构化的文档协作工作流
- 三阶段方法：上下文收集 → 精炼结构 → 读者测试
- 适合各种文档类型（PRD、技术规范、决策文档）

**触发关键词**：
- "写文档"、"起草文档"
- "技术规范"、"设计文档"
- "RFC"、"决策文档"

**工作流程**：
```
Stage 1: 上下文收集
  └─ Claude提问，你回答
     目标：缩小信息差距

Stage 2: 精炼和结构化
  └─ 逐节构建
     ├─ 头脑风暴
     ├─ 起草
     └─ 编辑

Stage 3: 读者测试
  └─ 用新的Claude实例测试
     目标：发现盲点
```

**适用场景**：
- ✅ 需要深度协作的文档
- ✅ 文档需要多次迭代
- ✅ 需要确保文档对读者友好
- ✅ 复杂的技术规范或设计文档

```

  - 用户名输入框
  - 密码输入框
  - 忘记密码链接

  - 用户名格式验证（邮箱/手机号）
  - 密码复杂度要求（8位以上，包含字母数字）
  - 登录失败处理（5次后锁定30分钟）
  - 记住我功能（7天免登录）

生成基础PRD + 待确认问题
```
